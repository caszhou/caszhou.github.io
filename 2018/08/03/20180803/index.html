Component B

首先我们规定，在特殊的业务场景，我们需要这么去做。首先页面A会有一个初始值change，等于false，其次组件B会有一个change的副本aryChange，初始值和页面A的一样。

需要实现，改变aryChange的值为true，并执行do方法。上面的代码确实改变了aryChange的值，但是并没有执行do方法。

有很多种改法，我给出四种。

  1. 页面A直接改变组件aryChange，并去掉组件B的watch
  2. 组件B的do方法又watch到，再又watch去调用
  3. 在页面A调用组件B do方法的时候，在外套一个setTimeout
  4. 在页面A调用组件B do方法的时候，在外套一个this.$nextTick

在这里我们讨论一下 setTimeout 和 $nextTick

setTimeout

setTimeout只是异步执行组件B的do方法，那这个异步需要什么时候执行呢？1S 2S 10M？？？这个程序是无法控制，所以方案3是不靠谱的~

Event Loop

刚刚讲到js的异步，大家都知道js是单线程，单线程是如何做到异步？答案是Event Loop！

啥是EventLoop，上一张非常具体、形象的图。

具体的就不详细解释了。要了解的童鞋，上这儿看

$nextTick

知道了JS的内存模型，我们看下方案4，方案4用到了nextTick，最终要达到的效果就是异步调用组件B的do方法，先剧透下，这个方案是靠谱的，就是说在调用组件B的do方法的时候，aryChange肯定等于true了，具体原理是啥，了解一下?

我们看下官网API对此方法的说明

首先我们要知道vue是异步更新dom的，并且出于性能的考虑，event（vue的中心思想就是数据驱动视图）

对于组件B，change发生的变更，需要更新视图，随后aryChange发生变更，也需要更新视图，两者更新视图的event会合并成一次视图更新，所以当视图（dom）更新完后，aryChange的值肯定为true，这个时间再调用组件B的do方法是没有问题的。
